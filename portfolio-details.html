담당 역할

링크

요구사항

  • 아두이노와 STM32 보드를 여러 개 사용하여 링크를 구성하고 센싱 및 결과 출력을 하는 시스템을 설계하는 것을 목표
  • 아두이노 Atmega328p의 레지스터 코딩, STM32의 부분 레지스터 코딩을 이용
  • 링크구성에 통신수단인 UART, USART, I2C, SPI, CAN 등의 여러가지 통신을 이용하여 조별 37종 센서를 지급받아 많은 센서를 사용하는 것을 목표

활동 기간

2024.09.07 ~ 2024.10.18

담당 역할

  • 팀장
  • 시스템 아키텍처 설계
  • 세탁 모드 설정 SW 컴포넌트(Atmega328P) 개발
  • 디스플레이 SW 컴포넌트(STM32 F103RB) 개발

요구사항 분석 및 인터페이스 정의

디스플레이 SW 컴포넌트(STM32 F103RB)

  • 64Mhz Clock을 이용해서 Prescaler를 7999로 설정.
  • 64Mhz/7999+1로 Counter Clock 설정.
  • 8000hz로 Counter의 주기를 9999로 가져가 1.25초 대략 1초마다 Timer Overflow Interrupt가 발생시키도록 설정.

Trouble Shooting

switch (state)
{
  case WASHING :
    laundryData[0] = encoderValue;
    SPI_sendDataToSTM(sendingStateData[1]);
    _delay_ms(10);
    if (ackSTM == 0x20)
    {
      SPI_sendDataToAr(encoderValue);
      state = WATERTEMP;
    }
    break;
  case WATERTEMP :
    laundryData[1] = encoderValue;
    SPI_sendDataToSTM(sendingStateData[2]);
    _delay_ms(10);
    if (ackSTM == 0x20)
    {
      SPI_sendDataToAr(encoderValue);
      state = RINSECOUNT;
    }
    break;
}

세탁 모드 설정 SW 컴포넌트(Atmega328P) 코드

void HAL_SPI_RxCpltCallback(
    SPI_HandleTypeDef *hspi)
{
  if (hspi->Instance == SPI1)
  {
    if (receivedData < 3)
      memcpy(&encoderData,
        &receivedData,
        sizeof(receivedData));
    else if (receivedData >= 3)
    {
      memcpy(&stateData,
        &receivedData,
        sizeof(receivedData));
      ack = 0x20;
      HAL_SPI_Transmit(
        &hspi1, &ack, 1, 1);
    }
    receivedFlag = 1;
  }
}

디스플레이 SW 컴포넌트(STM32 F103RB) SPI Callback 코드

아두이노 로타리의 버튼을 누를 시에 Data와 아두이노 로타리의 회전에 의한 Data가 겹쳐서 씹히는 경우가 발생.
이를 방지한 버튼을 눌렀을 시의 Data를 잘 받았으면 ack = 0x20을 전송.
